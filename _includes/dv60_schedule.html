{%- comment -%} DV60 programme — pure-Liquid, no JS. Reads _data/dv60-schedule.yml and joins talks to _data/dv60-abstracts.yml by slug for title + link. Drop {% include dv60_schedule.html %} into dv60.html, and load the component styles from that page's front matter: extra_css: ["/assets/css/dv60-schedule.css?v=1"] Layout: the DOM is day-major (each day's header + cards are emitted together), but every card carries an explicit grid-row (its time slot) and grid-column (its day), so on DESKTOP CSS Grid lays them out as a time-aligned matrix — same-time cards share a row and auto-size to equal height. Breaks are emitted as separate elements that span grid-column 1 / span k, where k = the number of days that have that break (so 10:30 spans all 3 columns; lunch / 15:30 span 2, leaving Wednesday — morning only — blank). On MOBILE the grid container becomes display:block, the grid placement goes inert, and cards simply flow in day-major DOM order (Mon's day, then Tue, then Wed). Breaks are hidden on mobile (as in CT's no-JS fallback). Item types: talks have slug (-> abstract) or name (no abstract page; shows the item's inline `title` if given, else "TBA"); socials have `until`; breaks live in the per-day `breaks:` list.
